  CDN Scanner v0.0.2 — Major Enhancements (2026-05-05)
════════════════════════════════════════════════════════════════════════════════

SUMMARY
-------
Major stability and UX improvements. Added "Use Top 3 DNS" button, Resume
functionality, and Stage 3 chunking for large scans. Fixed 8 critical bugs
including JavaScript syntax error, false DNS poisoning, and Mode 1 config format.

NEW FEATURES (v0.0.2)
---------------------
✅ "Use Top 3 DNS" button — Auto-fills best DNS servers after E2E test
✅ Resume button — Continue stopped scans without re-entering settings (orange button)
✅ Stage 3 chunking — Process 600+ targets in batches of 25 for stability
   • Memory stays at ~300MB (was growing unbounded)
   • Shows "Batch X/Y" progress logs
   • Clean early-stop between batches

CORE FEATURES (v0.0.1)
----------------------
✅ Two scanning modes (Netlify/Vercel domain fronting + CloudFront IP rotation)
✅ Real xray-core validation (no false positives)
✅ 4-stage pipeline: Parse → TCP+TLS → DNS fallback → xray validation
✅ CIDR expansion (supports /16 blocks = 65,536 IPs)
✅ DNS censorship bypass (custom DNS + DoH fallback)
✅ Web UI with live progress, file import, stop button, export
✅ CLI interface with full pipeline support
✅ Parallel processing (50 TCP workers + 5 xray workers)

CRITICAL FIXES (v0.0.2)
-----------------------
✅ JavaScript syntax error — Fixed `dnsMode E2E:` → `dnsModeE2E:` (broke all UI)
✅ False poisoning detection — Added European Google IP ranges (64.233.x.x, 74.125.x.x)
✅ 8.8.8.8 HTTP 400 error — Fixed DoH endpoint (https://dns.google/resolve)
✅ Mode 1 domain preservation — Config now uses domain (not IP) as address
   • Example: @docusign.com:443 (was @15.197.167.90:443)
   • More resilient (survives IP blocks)
✅ Force Custom DNS bypass — Checkbox now actually works (domains skip Stage 1)
✅ UI clarity for Mode 1 — Shows "domain (IP)" format in results
✅ DNS validation — Rejects invalid inputs (e.g., "xxxxx")
✅ Unreliable latency — Changed quick_test=False to eliminate false positives

KNOWN LIMITATIONS
-----------------
• IPv6 not supported (IPv4 only)
• Resume only works in same session (lost on app restart)
• Memory-only storage (scans lost on server restart)
• CIDR capped at 65,536 IPs (prevents accidental /8 expansions)
• Port 5000 conflict if running 2 instances (close first before opening second)
• Very slow DNS servers (>5s) marked as timeout (false negative)
• xray zombie processes if Python crashes (kill manually with Task Manager)

QUICK START
-----------
1. Standalone .exe (No Python Needed):
   dist\CDN-Scanner.exe
   # Browser opens at http://localhost:5000

2. Python Web Interface:
   python main.py
   # Open http://localhost:5000

3. CLI Scanner:
   python scanner.py "vless://your-link" --mode mode1 --max 10

4. Rebuild .exe (after changes):
   python -m PyInstaller build.spec --clean --noconfirm
